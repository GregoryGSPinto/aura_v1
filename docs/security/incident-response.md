# Incident Response

## Runbooks prioritarios

- segredo comprometido
- abuso de tool
- token comprometido
- bypass de autorizacao
- prompt injection confirmado
- dependencia critica vulneravel

## Passos base

1. conter a superficie afetada
2. revogar credenciais e integracoes
3. preservar evidencias e request IDs
4. revisar audit logs e escopo do incidente
5. aplicar correcao e validar rollback/forward fix
